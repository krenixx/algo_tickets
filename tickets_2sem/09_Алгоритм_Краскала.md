## Алгоритм Краскала

### Идея

Жадный алгоритм построения минимального остовного дерева (MST) в связном неориентированном взвешенном графе.
Алгоритм выбирает ребра в порядке увеличения их веса и добавляет ребро, если оно не образует цикл с уже выбранными. Проверка на образование цикла работает на системе непересекающихся множеств (DSU - Disjoint set union).     

#### Теория

**Минимальное остовное дерево (MST)** - это ациклический связный подграф, в который 
входят все его вершины, обладающий минимальным суммарным весом ребер.

**Лемма о безопасном ребре**


### Принцип работы

1. Сортируем список ребер по возрастанию весов.
2. Инициализировать DSU, где каждая вершина в отдельном множестве.
3. Проходить по ребрам в порядке сортировки:
    - Для ребра `(u, v, w)` проверить, принадлежат ли `u` и `v` одному множеству: `find(u) == find(v)`.
    - Если нет: добавить ребро в MST и выполнить `unite(u, v)` (объеденить множества).
    - Если да: пропустить.
4. Повторять пункт 3, пока в MST не будет `n-1` ребер.

### Сложность

1. Временная: 
    - Сортировка `O(ElogE)`
    - Операции в DSU: `O(E * α(V))`, где `α(V)` - обратная функция Аккермана.
    Итого: `O(ElogE)`
2. Пространственная:
    - Список ребер `O(E)`
    - DSU `O(V)`
    Итого: `O(V + E)`

### Реализация на C++

```cpp
struct DSU {
    int n;
    std::vector<int> parent;
    std::vector<int> rankv;

    DSU(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;
        parent.resize(n + 1);
        rankv.assign(n + 1, 0);
        for (int i = 1; i <= n; ++i) {
            parent[i] = i;
        }
    } 

    int find(int x) {
        if (parent[x] == x) return x;
        return parent[x] = find(parent[x]);  // сжатие пути
    }

    bool unite(int a, int b) {
        a = find(a);
        b = find(b);
        if (a == b) return false;
        if (rankv[a] < rankv[b]) {
            std::swap(a, b);
        }
        parent[b] = a;
        if (rankv[a] == rankv[b]) {
            rankv[a]++;
        }
        return true;
    }
};

struct Edge {
    int u, v;
    long long w;
    int idx;
};

int main() {
    int n, m;
    std::vector<Edge> edges;
    edges.reserve(m);
    for (int i = 0; i < m; ++i) {
        int u, v;
        long long w;
        std::cin >> u >> v >> w;
        edges.push_back({u, v, w, i});
    }

    sort(edges.begin(), edges.end(), [](const Edge& a, const Edge& b) {
        if (a.w != b.w) return a.w < b.w;
        return a.idx < b.idx;
    });

    DSU dsu(n);
    std::vector<Edge> mst_edges;
    mst_edges.reserve(n - 1);
    long long total_weight = 0;

    for (const auto& e : edges) {
        if (dsu.unite(e.u, e.v)) {
            mst_edges.push_back(e);
            total_weight += e.w;
            if ((int)mst_edges.size() == n - 1) break;
        }
    }

    if ((int)mst_edges.size() != n - 1) {
        std::cout << "no MST" << "\n";  // граф несвязен, MST нет
    }

    std::cout << total_weight << '\n';
    for (const auto& e : mst_edges) {
        std::cout << e.u << ' ' << e.v << ' ' << e.w << "\n";  
    }
}
```

`parent` — массив размера `n`, где `parent[x]` хранит «родителя» вершины x в дереве, представляющем её множество.
Если `parent[x] == x`, то `x` — корень (репрезентант) своего множества.
Изначально для n элементов обычно `parent[i] = i` — каждый элемент в своём множестве.

`rankv` — вспомогательный массив, который хранит «ранг» корня множества. Интерпретация: в классической реализации `rank` — это приблизительная верхняя оценка высоты дерева (не обязательно точная высота).
Иногда вместо `rank` используют `size` (размер множества). Тогда при объединении присоединяют меньшую по размеру в большую.
Цель `rank`/`size` — поддержать дерево достаточно «плоским», чтобы операции были быстрыми.
Идея: при объединении (`union`) мы присоединяем корень одного дерева под корень другого так, чтобы минимизировать рост высоты. rankv помогает выбрать, какой корень сделать родителем.

`Сжатие пути` — оптимизация для операции `find(x)`. Когда мы ищем корень вершины x, мы рекурсивно или итеративно поднимаемся к корню. Сжатие пути делает все узлы на этом пути непосредственными детьми корня.
После сжатия путь от этих узлов к корню становится длиной 1, что ускоряет последующие операции.
