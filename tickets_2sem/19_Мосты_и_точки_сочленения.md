# Мосты и точки сочленения

## Определения

**Мост** — ребро $(u, v)$, удаление которого увеличивает число компонент связности графа.

**Точка сочленения** — вершина $v$, удаление которой (вместе со всеми инцидентными рёбрами) увеличивает число компонент связности графа.

---

## Вспомогательные величины

Оба алгоритма работают на основе DFS и используют два массива:

- `tin[v]` — **время входа**: порядковый номер вершины $v$ при первом посещении.
- `low[v]` — **минимальное время входа**, достижимое из поддерева $v$ по одному обратному ребру:

$$\text{low}[v] = \min\begin{cases} \text{tin}[v] \\ \text{tin}[w] & \text{для всех обратных рёбер } (v, w) \\ \text{low}[u] & \text{для всех } u \text{ — детей } v \text{ в дереве DFS} \end{cases}$$

Интуиция: `low[v]` показывает, «как высоко» может прыгнуть поддерево $v$ по обратным рёбрам. Если прыгнуть выше родителя нельзя — родительское ребро является мостом.

---

## Нахождение мостов

### Критерий

Ребро $(u, v)$, где $v$ — сын $u$ в дереве DFS, является мостом тогда и только тогда, когда:

$$\text{low}[v] > \text{tin}[u]$$

Это означает: из поддерева $v$ нет обратного ребра, ведущего в $u$ или выше. Значит, единственный путь из поддерева $v$ в остальной граф — через ребро $(u, v)$.

### Кратные рёбра

Наивная проверка «не идём обратно в родителя» ломается при кратных рёбрах: два ребра между $u$ и $v$ образуют цикл, значит ни одно из них не мост. Решение — передавать **идентификатор ребра**, а не вершину родителя.

### Реализация

```cpp
#include <vector>
#include <algorithm>
using namespace std;

struct BridgeFinder {
    int n, timer = 0;
    vector<vector<pair<int,int>>> g;  // g[v] = {(to, edge_id)}
    vector<int> tin, low;
    vector<bool> visited;
    vector<pair<int,int>> bridges;

    BridgeFinder(int n) : n(n), g(n), tin(n), low(n), visited(n, false) {}

    void add_edge(int u, int v, int id) {
        g[u].push_back({v, id});
        g[v].push_back({u, id});
    }

    void dfs(int v, int parent_edge) {
        visited[v] = true;
        tin[v] = low[v] = timer++;

        for (auto [to, eid] : g[v]) {
            if (eid == parent_edge) continue;     // не идём по тому же ребру обратно
            if (visited[to]) {
                low[v] = min(low[v], tin[to]);    // обратное ребро
            } else {
                dfs(to, eid);
                low[v] = min(low[v], low[to]);
                if (low[to] > tin[v])             // критерий моста
                    bridges.push_back({v, to});
            }
        }
    }

    vector<pair<int,int>> find_bridges() {
        for (int i = 0; i < n; i++)
            if (!visited[i]) dfs(i, -1);
        return bridges;
    }
};
```

## Нахождение точек сочленения

### Критерий

Вершина $u$ является точкой сочленения в одном из двух случаев:

**Случай 1 — $u$ не корень DFS-дерева:**

$$\exists \text{ сын } v : \text{low}[v] \ge \text{tin}[u]$$

Поддерево $v$ не может «перепрыгнуть» выше $u$ — значит, при удалении $u$ оно отсоединится.

> Заметьте: для мостов условие строгое ($>$), для точек сочленения — нестрогое ($\ge$). Если $\text{low}[v] = \text{tin}[u]$, поддерево $v$ достигает $u$, но не выше — $u$ всё равно точка сочленения, хотя ребро $(u,v)$ мостом не является.

**Случай 2 — $u$ является корнем DFS-дерева:**

$$u \text{ — точка сочленения} \iff u \text{ имеет } \ge 2 \text{ детей в дереве DFS}$$

Корень особый: у него нет родителя, поэтому обратные рёбра из поддеревьев никогда не ведут «выше» него. Корень разделяет граф тогда и только тогда, когда у него несколько независимых поддеревьев.

### Реализация

```cpp
struct ArticulationFinder {
    int n, timer = 0;
    vector<vector<int>> g;
    vector<int> tin, low;
    vector<bool> visited, is_cut;

    ArticulationFinder(int n)
        : n(n), g(n), tin(n), low(n), visited(n, false), is_cut(n, false) {}

    void add_edge(int u, int v) {
        g[u].push_back(v);
        g[v].push_back(u);
    }

    void dfs(int v, int parent) {
        visited[v] = true;
        tin[v] = low[v] = timer++;
        int children = 0;

        for (int to : g[v]) {
            if (to == parent) continue;
            if (visited[to]) {
                low[v] = min(low[v], tin[to]);   // обратное ребро
            } else {
                children++;
                dfs(to, v);
                low[v] = min(low[v], low[to]);

                // случай 1: v не корень
                if (parent != -1 && low[to] >= tin[v])
                    is_cut[v] = true;
            }
        }

        // случай 2: v — корень
        if (parent == -1 && children >= 2)
            is_cut[v] = true;
    }

    vector<int> find_articulations() {
        for (int i = 0; i < n; i++)
            if (!visited[i]) dfs(i, -1);
        vector<int> result;
        for (int i = 0; i < n; i++)
            if (is_cut[i]) result.push_back(i);
        return result;
    }
};
```

> **Осторожно с кратными рёбрами** в задаче на точки сочленения: фильтрация `if (to == parent)` пропускает первое встреченное ребро к родителю, но при кратных рёбрах второе ребро законно использовать как обратное. Аналогично мостам, надёжнее фильтровать по идентификатору ребра.

## Пример

```
Граф: 0—1—2—3
          |
          4
```

DFS от 0, порядок: 0→1→2→3, возврат, 1→4.

| $v$ | `tin` | `low` |
|-----|-------|-------|
| 0 | 0 | 0 |
| 1 | 1 | 1 |
| 2 | 2 | 2 |
| 3 | 3 | 3 |
| 4 | 4 | 4 |

**Мосты:** все рёбра — `low[сына] > tin[родителя]` для каждого.

**Точки сочленения:**
- Вершина 1: `low[2] = 2 >= tin[1] = 1` ✓, `low[4] = 4 >= tin[1] = 1` ✓ → точка сочленения.
- Вершина 2: `low[3] = 3 >= tin[2] = 2` ✓ → точка сочленения.
- Вершины 0, 3, 4 — нет.

---

## Сравнение критериев

| | Мост $(u \to v)$ | Точка сочленения $u$ (не корень) |
|---|---|---|
| Условие | $\text{low}[v] > \text{tin}[u]$ | $\text{low}[v] \ge \text{tin}[u]$ |
| Смысл | поддерево $v$ не достигает $u$ | поддерево $v$ не поднимается выше $u$ |

---

## Сложность

| | Значение |
|---|---|
| Время | $O(V + E)$ |
| Память | $O(V + E)$ |

Один проход DFS — линейное время.

## Типичные вопросы

**В.** Почему для точки сочленения условие нестрогое ($\ge$), а для моста строгое ($>$)?  
**О.** Если $\text{low}[v] = \text{tin}[u]$, то из поддерева $v$ есть обратное ребро прямо в $u$ — ребро $(u,v)$ не мост (есть цикл). Но $u$ всё равно точка сочленения: убрав её, поддерево $v$ отсоединится, ведь обратное ребро ведёт именно в $u$, а не выше.

**В.** Почему корень рассматривается отдельно?  
**О.** У корня нет родителя, поэтому обратные рёбра из любого его поддерева никогда не ведут «выше» — стандартный критерий всегда бы давал «точка сочленения». Реальный критерий: если у корня $\ge 2$ детей в DFS-дереве, то при его удалении они разъединяются.

**В.** Как корректно обработать кратные рёбра?  
**О.** Передавать идентификатор ребра (а не вершину родителя) и пропускать ровно то ребро, по которому пришли — тогда второе кратное ребро законно учитывается как обратное.
