# Алгоритм КМП и Автомат КМП

## 1. Алгоритм Кнута–Морриса–Пратта (КМП)

### Задача

Найти все вхождения паттерна $p$ (длина $m$) в текст $s$ (длина $n$) за $O(n + m)$.

### Идея

Строим префикс-функцию для строки t = p + # + s.

- Разделитель `#` — символ, не встречающийся ни в $p$, ни в $s$.
- Это гарантирует $\pi[i] \le m$ для всех $i \ge m + 1$.
- Если $\pi[i] = m$ — паттерн найден; позиция вхождения в $s$: $i - 2m$.

### Реализация (через префикс-функцию)

```cpp
#include <vector>
#include <string>
using namespace std;

vector<int> prefix_function(const string& s) {
    int n = s.size();
    vector<int> pi(n, 0);
    for (int i = 1; i < n; i++) {
        int k = pi[i - 1];
        while (k > 0 && s[k] != s[i])
            k = pi[k - 1];
        if (s[k] == s[i]) k++;
        pi[i] = k;
    }
    return pi;
}

vector<int> kmp_search(const string& text, const string& pat) {
    string t = pat + "#" + text;
    auto pi = prefix_function(t);
    int m = pat.size();
    vector<int> occ;
    for (int i = m + 1; i < (int)t.size(); i++)
        if (pi[i] == m)
            occ.push_back(i - 2 * m);   // позиция в text
    return occ;
}
```

### Сложность

| Этап | Время |
|------|-------|
| Построение $\pi$ | $O(n + m)$ |
| Обход результатов | $O(n)$ |
| **Итого** | $O(n + m)$ |

## 2. Автомат КМП

### Мотивация

Обычный КМП обрабатывает **фиксированный текст**. Но что если текст поступает **потоком** (посимвольно), и нам нельзя хранить его целиком? Или нужно искать несколько раз по разным текстам с одним паттерном?

Решение — заранее построить **детерминированный конечный автомат (ДКА)** по паттерну.

### Что такое автомат КМП

- **Состояния:** $0, 1, \ldots, m$ — означают «сопоставлено $k$ символов паттерна».
- **Начальное состояние:** $0$.
- **Принимающее состояние:** $m$ (паттерн найден).
- **Переходы:** $\delta(q, c)$ — из состояния $q$ по символу $c$ попадаем в состояние $\delta(q, c)$.

### Как определить переходы $\delta(q, c)$

Находясь в состоянии $q$, мы сопоставили $p[0..q-1]$. Читаем символ $c$:

- Если $p[q] = c$: переходим в $q + 1$.
- Иначе: нам нужно найти наибольший $k < q$ такой, что $p[0..k-1]$ — суффикс $p[0..q-1] + c$ и $p[k] = c$.

Это в точности то, что делает откат по цепочке $\pi$. Поэтому:

$$\delta(q, c) = \pi\text{-}{\rm extension}(q, c)$$

### Алгоритм построения за $O(m \cdot |\Sigma|)$

```cpp
// Построение таблицы переходов автомата КМП
// sigma — размер алфавита (например, 26 для строчных букв)
// Символ c кодируется как c - 'a'

vector<vector<int>> build_automaton(const string& pat, int sigma = 26) {
    int m = pat.size();
    // delta[q][c] — переход из состояния q по символу c
    vector<vector<int>> delta(m + 1, vector<int>(sigma, 0));

    // Вычисляем префикс-функцию для паттерна
    auto pi = prefix_function(pat);

    for (int q = 0; q <= m; q++) {
        for (int c = 0; c < sigma; c++) {
            if (q < m && pat[q] - 'a' == c) {
                // Совпадение — идём вперёд
                delta[q][c] = q + 1;
            } else if (q == 0) {
                // Откатываться некуда
                delta[q][c] = 0;
            } else {
                // Используем уже вычисленный переход из состояния pi[q-1]
                delta[q][c] = delta[pi[q - 1]][c];
            }
        }
    }
    return delta;
}
```

> **Ключевая оптимизация:** при вычислении $\delta(q, c)$ когда $p[q] \ne c$, мы используем уже посчитанный $\delta[\pi[q-1]][c]$. Это работает, потому что $\pi[q-1]$ — длина наибольшего собственного префикса-суффикса $p[0..q-1]$, и автомат для него уже построен.

### Поиск с помощью автомата

```cpp
vector<int> kmp_automaton_search(const string& text, const string& pat) {
    auto delta = build_automaton(pat);
    int m = pat.size();
    vector<int> occ;
    int q = 0;                              // текущее состояние
    for (int i = 0; i < (int)text.size(); i++) {
        q = delta[q][text[i] - 'a'];       // переход по символу
        if (q == m)
            occ.push_back(i - m + 1);      // паттерн найден
    }
    return occ;
}
```

### Обработка потока

```cpp
// Пример: символы приходят один за другим
struct KMPStream {
    vector<vector<int>> delta;
    int m, q = 0;
    vector<int> occ;

    KMPStream(const string& pat) : delta(build_automaton(pat)), m(pat.size()) {}

    void feed(char c) {
        q = delta[q][c - 'a'];
        if (q == m) {
            // зафиксировали вхождение — позицию можно считать снаружи
        }
    }

    bool matched() const { return q == m; }
};
```

## 3. Сравнение: КМП vs Автомат КМП

| | КМП (с $\pi$) | Автомат КМП |
|---|---|---|
| Предобработка | $O(m)$ | $O(m \cdot \|\Sigma\|)$ |
| Поиск | $O(n)$ | $O(n)$ |
| Память | $O(m)$ | $O(m \cdot \|\Sigma\|)$ |
| Потоковый ввод | неудобно | естественно |
| Несколько текстов | нужно повторять | строим автомат один раз |
| Простота реализации | проще | чуть сложнее |

## 4. Формальное описание автомата

Автомат КМП — ДКА:

$$M = (Q,\ \Sigma,\ \delta,\ q_0,\ F)$$

- $Q = \{0, 1, \ldots, m\}$ — состояния
- $\Sigma$ — алфавит
- $\delta: Q \times \Sigma \to Q$ — таблица переходов (построена выше)
- $q_0 = 0$ — начальное состояние
- $F = \{m\}$ — единственное принимающее состояние

Автомат принимает строку тогда и только тогда, когда она **оканчивается** вхождением $p$.

### Пример автомата для `p = "abab"`, $\Sigma = \{a, b\}$

```
Состояние | Совпадено | δ(·, a) | δ(·, b)
----------+-----------+---------+--------
    0     |    ""     |    1    |    0
    1     |    "a"    |    1    |    2
    2     |    "ab"   |    3    |    0
    3     |    "aba"  |    1    |    4
    4     |   "abab"  |    3    |    0   ← принимающее
```

Трассировка `text = "aababab"`:

```
i:  0  1  2  3  4  5  6
c:  a  a  b  a  b  a  b
q:  1  1  2  3  4  3  4  ← q=4 на i=4 и i=6 → вхождения на позициях 1 и 3
```

---

## 5. Связь автомата с $\pi$

Переход $\delta(q, c)$ при $p[q] \ne c$ — это откат, который в обычном КМП выполняется через цепочку $\pi$. Автомат просто **кэширует** результат этого отката в таблице, чтобы не делать его каждый раз при обработке текста.

## 6. Типичные вопросы

**В.** Почему сложность построения автомата $O(m \cdot |\Sigma|)$?  
**О.** Для каждого из $m+1$ состояний вычисляем $|\Sigma|$ переходов за $O(1)$ каждый (используя $\delta[\pi[q-1]][c]$, уже посчитанный).

**В.** Почему $\delta(q, c) = \delta[\pi[q-1]][c]$ при $p[q] \ne c$?  
**О.** После «сброса» мы оказываемся в состоянии $\pi[q-1]$ — наибольшем префиксе паттерна, который является суффиксом текущего прочитанного текста. Дальнейший переход по $c$ из этого состояния уже вычислен.

**В.** Чем автомат лучше при потоковом вводе?  
**О.** Переход $\delta(q, c)$ — $O(1)$ без каких-либо откатов. Каждый символ обрабатывается ровно одним табличным переходом.

**В.** Что произойдёт после попадания в состояние $m$?  
**О.** Автомат продолжает работу — из состояния $m$ тоже определены переходы. Это позволяет искать **перекрывающиеся** вхождения.
