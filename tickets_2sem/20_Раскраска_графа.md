# Жадная раскраска графа

## Задача

Раскрасить вершины графа в минимальное число цветов так, чтобы никакие две смежные вершины не имели одинаковый цвет. Это задача о **хроматическом числе** $\chi(G)$.

> Жадный алгоритм не гарантирует оптимальное решение — он даёт хорошее приближение. Точный поиск $\chi(G)$ — NP-трудная задача.

---

## Идея

Обходим вершины в порядке убывания степени. Каждой вершине присваиваем **наименьший цвет, не занятый её уже раскрашенными соседями**.

Эвристика «сначала вершины с большей степенью» (алгоритм **Welsh–Powell**) даёт лучший результат, чем произвольный порядок: высокостепенные вершины труднее раскрасить, поэтому их лучше обработать раньше, пока цветов использовано мало.

---

## Алгоритм

1. Отсортировать вершины по убыванию степени.
2. Всем вершинам присвоить цвет $-1$ («не раскрашена»).
3. Для каждой вершины $u$ в этом порядке:
   - собрать множество цветов, уже занятых соседями $u$;
   - присвоить $u$ минимальный неотрицательный цвет, не входящий в это множество.

---

## Реализация на C++

```cpp
#include <vector>
#include <algorithm>
#include <unordered_set>
using namespace std;

// graph — список смежности, V — число вершин
vector<int> greedy_coloring(const vector<vector<int>>& graph, int V) {
    // сортируем вершины по убыванию степени
    vector<int> order(V);
    iota(order.begin(), order.end(), 0);
    sort(order.begin(), order.end(), [&](int a, int b) {
        return graph[a].size() > graph[b].size();
    });

    vector<int> color(V, -1);

    for (int u : order) {
        // собираем занятые цвета соседей
        unordered_set<int> used;
        for (int nb : graph[u])
            if (color[nb] != -1)
                used.insert(color[nb]);

        // ищем минимальный свободный цвет
        int c = 0;
        while (used.count(c)) c++;
        color[u] = c;
    }

    return color;
}
```

## Пример

Граф: $V = 5$, рёбра $\{0{-}1,\ 0{-}2,\ 1{-}2,\ 1{-}3,\ 3{-}4\}$

```
Степени: 0→2, 1→3, 2→2, 3→2, 4→1
Порядок обхода: 1, 0, 2, 3, 4

u=1: соседи {0,2,3}, все -1 → цвет 0
u=0: соседи {1,2},   занято {0} → цвет 1
u=2: соседи {0,1},   занято {0,1} → цвет 2
u=3: соседи {1,4},   занято {0} → цвет 1
u=4: соседи {3},     занято {1} → цвет 0
```

Итог: `color = [1, 0, 2, 1, 0]` — использовано **3 цвета**.

## Сложность

| | Сложность |
|---|---|
| Время | $O(V \log V + (V + E))$ |
| Память | $O(V + E)$ |

- $O(V \log V)$ — сортировка вершин.
- $O(V + E)$ — обход каждой вершины и её соседей суммарно.
- Множество `used` на каждом шаге — не более $\deg(u)$ элементов.

---

## Свойства и ограничения

**Верхняя оценка:** алгоритм использует не более $\Delta(G) + 1$ цветов, где $\Delta(G)$ — максимальная степень вершины. Теорема Брукса уточняет: $\chi(G) \le \Delta(G)$ для графов, не являющихся полными или нечётными циклами.

**Зависимость от порядка:** результат зависит от порядка обхода. Один и тот же граф при разных перестановках вершин может быть раскрашен в разное число цветов.

**Не оптимален:** жадный алгоритм может использовать больше цветов, чем $\chi(G)$. Пример — граф, где жадность в плохом порядке даёт $n/2$ цветов вместо 2.

## Типичные вопросы

**В.** Почему сортируем по убыванию степени?  
**О.** Вершины с большой степенью имеют больше ограничений — их сложнее раскрасить. Обрабатывая их первыми, мы используем меньше цветов на старте, оставляя гибкость для вершин с меньшей степенью.

**В.** Когда жадный алгоритм даёт точный ответ?  
**О.** Для **хордальных графов** существует порядок (perfect elimination ordering), при котором жадность всегда находит $\chi(G)$.

**В.** Можно ли найти $\chi(G)$ точно эффективно?  
**О.** Нет — задача NP-трудная в общем случае. На практике используют точные алгоритмы с ветвями и границами или приближения.
