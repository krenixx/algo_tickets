## Алгоритм Куна для поиска максимального паросочетания в двудольном графе

### Теория

**Паросочетание** - подмножество ребер графа, в котором никакие два ребра несмежны друг другу (не имеют общей вершины).
**Максимальное пароочетание** - максимальное паросочетание, возможное для данного графа.
**Совершенное паросочетание** - паросочетание, покрывающее все вершины графа.
**Покрытая (насыщенная) вершина** - вершина, входящая в паросочетание.
**Свободная (ненасыщенная) вершина** - вершина, невходящая в паросочетание.

**Чередующийся путь** - путь, в котором ребра поочередно то принадлежат паросочетанию, то не принадледат ему.
**Увеличивающийся путь** - это чередующийся путь, который начинается и заканчивается в свободных вершинах. 
**Свойство**: в увеличивающимся пути ребер из паросочетания на 1 меньше, чем ребер не из него.

**Лемма Берджа**: паросочетание является максимальным тогда и только тогда, когда в графе не существует увеличивающегося пути.
То есть, если мы нашли такой путь, то можем "инвертировать" его ребра (заменить на ребро из паросочетания на ребро из вне и наоборот) и тогда максимальное паросочетание увеличится на 1.    

### Идея

Алгоритм ищет максимальное паросочетание в двудольном графе на основе увеличивающихся путей.

### Комментарий

Алгоритм Куна ищет увеличивающие пути простым поиском в глубину (DFS). В двудольном графе любой увеличивающий путь всегда будет переходить из одной доли в другую, и в нем никогда не возникнет циклов нечетной длины.

В произвольных (не двудольных) графах могут существовать «соцветия» (blossoms) - циклы нечетной длины. Если DFS попадет в такой цикл, он может не найти увеличивающий путь, даже если он есть. Для решения этой проблемы в произвольных графах используется более сложный алгоритм Эдмондса (сжатие цветков).

### Принцип работы

1. Для каждой вершины из первой доли графа пытаемся найти увеличивающийся путь через `DFS`:
    1. Пробуем соединить вершину `u` со свободным соседом `v`
    2. Если `v` занята `u'`, пытаемся рекурсивно найти для `u'` другой свободный вариант.
    3. Если удалось дойти до свободной вершины, "инвертируем ребра"(см теорию).
2. Количество успешных запусков `DFS` равно размеру максимального паросочетания.

### Асимптотика

Временная: `O(V * (V + E)) = O(V * E)`
Пространственная: `O(V)`

### Псевдокод

```python
# match[v] хранит индекс вершины из левой доли, с которой соединена ребром вершина v из правой
match = [-1] * RightPartSize
used = [] # Массив для пометки посещенных вершин в текущем DFS

def try_kuhn(u):
    if used[u]: return False
    used[u] = True
    
    for v in adj[u]: # v - вершина из правой доли
        # Если v свободна ИЛИ можем перенаправить того, кто с ней спарен
        if match[v] == -1 or try_kuhn(match[v]):
            match[v] = u # мэтчим u и v
            return True
            
    return False

# Основной цикл
count = 0
for i in range(LeftPartSize):
    used = [False] * LeftPartSize # Сбрасываем для каждого нового поиска
    if try_kuhn(i):
        count += 1
```
