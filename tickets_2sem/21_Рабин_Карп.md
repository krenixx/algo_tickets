# Алгоритм Рабина–Карпа

## 1. Задача поиска подстроки

Дана строка-текст $T$ длины $n$ и образец $P$ длины $m$.  
Найти все вхождения $P$ в $T$.

Наивный подход — $O(nm)$. Рабин–Карп даёт **среднее $O(n + m)$** за счёт хэширования.

---

## 2. Ключевая идея — полиномиальный хэш

Каждую строку длины $m$ представляем числом (хэшем):

$$h(s) = \left(\sum_{i=0}^{m-1} s[i] \cdot p^{m-1-i}\right) \bmod q$$

где:
- $p$ — основание (обычно простое, например **31** или **131**)
- $q$ — большой простой модуль (например **1 000 000 007**)

**Сравнение хэшей быстрее, чем посимвольное сравнение строк.**

---

## 3. Скользящий хэш (Rolling Hash)

Главная оптимизация: пересчёт хэша окна за $O(1)$.

При сдвиге окна с позиции $i$ на $i+1$:

$$h_{i+1} = \bigl(h_i - T[i] \cdot p^{m-1}\bigr) \cdot p + T[i+m]$$

всё по модулю $q$.

Это позволяет пройти по всем $n - m + 1$ окнам текста за $O(n)$.

## 4. Алгоритм (псевдокод)

```
1. Вычислить hash(P)
2. Вычислить hash(T[0..m-1])
3. Для каждого окна i от 0 до n-m:
     если hash(окно) == hash(P):
         проверить посимвольно (проверка)
         если совпало → вхождение найдено
     пересчитать hash для следующего окна (rolling hash)
```

> **проверка обязательна!** Хэш-коллизии дают ложные срабатывания.

---

## 5. Реализация на C++

```cpp
#include <iostream>
#include <string>
#include <vector>

// Поиск всех вхождений pattern в text.
// Возвращает вектор начальных индексов (0-based).
std::vector<int> rabinKarp(const std::string& text,
                           const std::string& pattern)
{
    std::vector<int> result;

    const long long P = 131;          // основание
    const long long MOD = 1e9 + 7;   // простой модуль

    int n = text.size();
    int m = pattern.size();
    if (m > n) return result;

    // Предвычисляем P^(m-1) mod MOD
    long long pPow = 1;
    for (int i = 0; i < m - 1; ++i)
        pPow = pPow * P % MOD;

    // Хэш образца
    long long patHash = 0;
    for (int i = 0; i < m; ++i)
        patHash = (patHash * P + pattern[i]) % MOD;

    // Хэш первого окна текста
    long long winHash = 0;
    for (int i = 0; i < m; ++i)
        winHash = (winHash * P + text[i]) % MOD;

    // Скользим по тексту
    for (int i = 0; i <= n - m; ++i) {
        if (winHash == patHash) {
            // проверка во избежание ложных срабатываний
            if (text.substr(i, m) == pattern)
                result.push_back(i);
        }

        // Пересчёт rolling hash
        if (i < n - m) {
            winHash = (winHash - (long long)text[i] * pPow % MOD + MOD) % MOD;
            winHash = (winHash * P + text[i + m]) % MOD;
        }
    }

    return result;
}

int main()
{
    std::string text    = "abracadabra";
    std::string pattern = "abra";

    auto positions = rabinKarp(text, pattern);

    std::cout << "Pattern \"" << pattern
              << "\" found at positions:";
    for (int pos : positions)
        std::cout << " " << pos;
    std::cout << "\n"; // вывод: 0 7

    return 0;
}
```

### Пример работы

| Окно | Текст | Хэш совпал? | Проверка |
|------|-------|-------------|-------------|
| 0    | `abra` | ✅ | ✅ → **вхождение** |
| 1    | `brac` | ❌ | — |
| 2    | `raca` | ❌ | — |
| ...  | ...   | ❌ | — |
| 7    | `abra` | ✅ | ✅ → **вхождение** |

## 6. Сложность

| Характеристика | Значение |
|----------------|----------|
| Предобработка | $O(m)$ |
| Поиск (среднее) | $O(n + m)$ |
| Поиск (худший случай) | $O(nm)$ — при коллизиях на каждом шаге |
| Память | $O(1)$ доп. памяти |

Худший случай возникает при большом числе ложных совпадений хэша — на практике крайне редко при хорошем выборе $p$ и $q$.

## 7. Множественный поиск (бонус)

Рабин–Карп особенно выгоден при поиске **нескольких образцов** одновременно:  
хэши всех образцов кладём в `unordered_set`, и за один проход по тексту проверяем всё.

```cpp
#include <unordered_set>

// patterns — набор образцов одинаковой длины m
std::vector<int> multiSearch(const std::string& text,
                             const std::vector<std::string>& patterns)
{
    if (patterns.empty()) return {};

    const long long P   = 131;
    const long long MOD = 1e9 + 7;

    int n = text.size();
    int m = patterns[0].size();

    // Хэшируем все образцы
    std::unordered_set<long long> patHashes;
    for (const auto& pat : patterns) {
        long long h = 0;
        for (char c : pat) h = (h * P + c) % MOD;
        patHashes.insert(h);
    }

    long long pPow = 1;
    for (int i = 0; i < m - 1; ++i) pPow = pPow * P % MOD;

    long long winHash = 0;
    for (int i = 0; i < m; ++i)
        winHash = (winHash * P + text[i]) % MOD;

    std::vector<int> result;
    for (int i = 0; i <= n - m; ++i) {
        if (patHashes.count(winHash)) {
            // здесь можно добавить точную верификацию
            result.push_back(i);
        }
        if (i < n - m) {
            winHash = (winHash - (long long)text[i] * pPow % MOD + MOD) % MOD;
            winHash = (winHash * P + text[i + m]) % MOD;
        }
    }
    return result;
}
```

## 8. Сравнение с другими алгоритмами

| Алгоритм | Препроцессинг | Поиск | Особенности |
|----------|---------------|-------|-------------|
| Наивный | $O(1)$ | $O(nm)$ | Прост, медленный |
| **Рабин–Карп** | $O(m)$ | $O(n+m)$ ср. | Хорошо для множественного поиска |
| КМП | $O(m)$ | $O(n+m)$ | Гарантия в худшем случае |
| Бойер–Мур | $O(m+\sigma)$ | $O(n/m)$ лучший | Быстрый на длинных образцах |

## 9. Частые вопросы на коллоквиуме

**Q: Зачем нужна проверка после совпадения хэша?**  
A: Из-за коллизий — разные строки могут давать одинаковый хэш. проверка гарантирует корректность.

**Q: Как выбрать $p$ и $q$?**  
A: $p$ — простое число, чуть больше размера алфавита (для ASCII: 131). $q$ — большое простое (10⁹+7). Можно использовать два разных модуля для уменьшения вероятности коллизий.

**Q: Когда Рабин–Карп лучше КМП?**  
A: При поиске нескольких образцов одновременно: один проход по тексту против $k$ независимых KMP-проходов.

**Q: Почему худший случай $O(nm)$?**  
A: Если каждое окно даёт хэш-коллизию, выполняется посимвольная проверка для каждого из $n-m+1$ окон.
