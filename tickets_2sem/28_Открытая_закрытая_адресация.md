# Открытая и закрытая адресация

## Закрытая адресация (Separate Chaining)

### Идея

Каждая ячейка таблицы — **список** элементов. При коллизии новый элемент просто добавляется в список.

```
индекс | содержимое
  0    | → [20] → [40] → null
  1    | → [11] → null
  2    | → null
  3    | → [3] → null
```

### Структура

```cpp
struct Node { int key, val; Node* next; };

struct ChainingMap {
    static const int M = 1009;
    Node* table[M] = {};

    int h(int k) { return ((k % M) + M) % M; }

    void insert(int k, int v) {
        int i = h(k);
        for (auto* p = table[i]; p; p = p->next)
            if (p->key == k) { p->val = v; return; }
        table[i] = new Node{k, v, table[i]};
    }

    int* find(int k) {
        for (auto* p = table[h(k)]; p; p = p->next)
            if (p->key == k) return &p->val;
        return nullptr;
    }
};
```

### Плюсы и минусы

| ✓ Плюсы | ✗ Минусы |
|---------|---------|
| Прост в реализации | Доп. память на указатели |
| $\alpha$ может быть $> 1$ | Плохая кэш-локальность |
| Удаление тривиально | Много аллокаций |

## Открытая адресация (Open Addressing)

### Идея

Все элементы — в самом массиве. При коллизии **ищем другую свободную ячейку** по некоторому правилу — **последовательности пробирования** $h(k, 0), h(k, 1), h(k, 2), \ldots$

Правило должно давать перестановку $[0, M)$.

---

## Линейное пробирование

$$h(k, i) = (h(k) + i) \bmod M$$

Смотрим ячейки подряд: $h(k), h(k)+1, h(k)+2, \ldots$

```cpp
struct LinearProbing {
    static const int M = 1009;
    static const int EMPTY = INT_MIN, DELETED = INT_MIN + 1;
    int keys[M], vals[M];

    LinearProbing() { fill(keys, keys + M, EMPTY); }

    int h(int k, int i) { return ((k % M) + M + i) % M; }

    void insert(int k, int v) {
        for (int i = 0; i < M; i++) {
            int j = h(k, i);
            if (keys[j] == EMPTY || keys[j] == DELETED || keys[j] == k) {
                keys[j] = k; vals[j] = v; return;
            }
        }
    }

    int* find(int k) {
        for (int i = 0; i < M; i++) {
            int j = h(k, i);
            if (keys[j] == EMPTY) return nullptr;
            if (keys[j] == k) return &vals[j];
        }
        return nullptr;
    }

    void erase(int k) {
        for (int i = 0; i < M; i++) {
            int j = h(k, i);
            if (keys[j] == EMPTY) return;
            if (keys[j] == k) { keys[j] = DELETED; return; }
        }
    }
};
```

### Метка DELETED

При удалении **нельзя** просто поставить EMPTY — это разорвёт цепочку поиска. Ставим специальную метку `DELETED` («надгробие»): поиск через неё продолжается, вставка может использовать её как свободную ячейку.

### Кластеризация

**Первичная кластеризация** — при линейном пробировании занятые ячейки образуют длинные непрерывные блоки («кластеры»). Чем больше кластер, тем больше вероятность, что следующий элемент попадёт в него → кластеры растут лавинообразно.

При $\alpha = 0.9$ среднее число зондирований при успешном поиске $\approx 5.5$, при неуспешном $\approx 50$.

## Квадратичное пробирование

$$h(k, i) = (h(k) + c_1 \cdot i + c_2 \cdot i^2) \bmod M$$

Обычно $c_1 = c_2 = 0.5$, $M$ — степень двойки. Или $c_1 = 0, c_2 = 1$, $M$ — простое.

```cpp
int h(int k, int i) {
    return (((k % M) + M) + i * i) % M;
}
```

Устраняет первичную кластеризацию, но возникает **вторичная** — ключи с одним $h(k)$ идут по одному пути.

**Гарантия покрытия:** при $M$ — простом и $c_1=c_2=1/2$ последовательность покрывает ровно $M/2$ различных ячеек. При $M$ — простом вида $4k+3$ и $c_2=1$, $c_1=0$ — покрывает все $M$ ячеек.

---

## Двойное хеширование

$$h(k, i) = (h_1(k) + i \cdot h_2(k)) \bmod M$$

Шаг пррбирование зависит от ключа — вторичной кластеризации нет.

Требования:
- $M$ — простое
- $h_2(k) \ne 0$ и $h_2(k) \not\mid M$ — иначе зацикливание. Обычно $h_2(k) = 1 + (k \bmod (M-1))$.

```cpp
int h1(int k) { return ((k % M) + M) % M; }
int h2(int k) { return 1 + ((k % (M - 1)) + M - 1) % (M - 1); }
int h(int k, int i) { return (h1(k) + i * h2(k)) % M; }
```

---

## Сравнение методов пробирования

| Метод | Кластеризация | Кэш | Покрытие |
|-------|--------------|-----|----------|
| Линейное | первичная (сильная) | отлично | всегда все $M$ |
| Квадратичное | вторичная (слабая) | хорошо | зависит от $M$ |
| Двойное | почти нет | плохо | все $M$ (при правильном $h_2$) |

---

## Открытая vs закрытая адресация

| | Закрытая (цепочки) | Открытая |
|---|---|---|
| Память | $> M$ (списки) | ровно $M$ ячеек |
| Макс. $\alpha$ | $> 1$ | $< 1$ (обычно $< 0.7$) |
| Удаление | просто | нужны надгробия |
| Кэш | плохо (указатели) | хорошо (массив) |
| Худший случай | $O(n)$ | $O(n)$ |
| Среднее | $O(1)$ | $O(1)$ |

## Типичные вопросы

**В.** Почему при удалении нельзя поставить EMPTY?  
**О.** Это разрывает цепочку пробирования — поиск следующих элементов остановится раньше времени.

**В.** При каком $\alpha$ стоит рехешировать?  
**О.** Для открытой адресации — при $\alpha > 0.7$, для цепочек — при $\alpha > 1$.

**В.** Что лучше на практике — линейное или двойное продировании?  
**О.** Линейное часто быстрее на практике из-за кэш-локальности, несмотря на кластеризацию.

**В.** Может ли квадратичное пробирование не найти свободную ячейку?  
**О.** Да, если $M$ выбрано неудачно. При $M$ — простом и $\alpha < 0.5$ свободная ячейка всегда найдётся.
